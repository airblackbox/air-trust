Privacy Policy

Last updated: April 7, 2026

The Short Version

AIR Blackbox is a local-first tool. Your code never leaves your machine. We don't collect, store, or transmit your source code, scan results, or compliance reports.

CLI Scanner (air-blackbox)

The AIR Blackbox CLI scanner runs 100% locally on your machine. It performs static analysis of your Python code against EU AI Act Articles 9 through 15. No code, file contents, or scan results are ever sent to any external server. All data stays on your local filesystem.

MCP Server

The AIR Blackbox MCP server runs locally within your IDE (Claude Desktop, Cursor, etc.). It provides the same scanning capabilities as the CLI. No data is transmitted externally by the MCP server itself.

Trust Layers

The optional trust layer packages (air-openai-trust, air-anthropic-trust, air-langchain-trust, etc.) intercept LLM API calls to add HMAC-SHA256 audit logging. Audit logs are stored locally in a SQLite database on your machine. The trust layers do not send data to AIR Blackbox servers.

Website (airblackbox.ai)

This website is a static site hosted on Vercel. We use Vercel Analytics for anonymous, aggregated page view counts. We do not use cookies for tracking. We do not collect personal information through the website unless you voluntarily contact us.

PyPI Distribution

AIR Blackbox packages are distributed through PyPI (the Python Package Index). Download counts are tracked by PyPI, not by us. We do not embed telemetry or analytics in any of our packages.

Third-Party Services

When you use trust layers with LLM providers (OpenAI, Anthropic, etc.), your prompts and responses are subject to those providers' privacy policies. AIR Blackbox does not control how those providers handle your data. The trust layers help you maintain local audit records of those interactions for compliance purposes.

Open Source

All AIR Blackbox code is open source under the Apache 2.0 license. You can audit the source code yourself to verify these privacy claims at github.com/air-blackbox.

Contact

For privacy questions, reach out via GitHub Issues.
